    移动端点项目有点杂，在这总结下！

    # 一、rem简单入门

    # 1、为啥要用rem

    • 开发移动端页面
    • 面对不同分辨率的手机
    • 面对不同尺寸的手机

    简单来说：rem就是指屏幕宽度的百分比，即n个rem等于可视区域的100%宽度 不包括高度

    实例分析： 1、设计师给750px宽度的设计图，（这里是位图像素） 2、假定将设计图宽等分为75份，也就是75rem=100%。那么1rem=10px
    3、这样你按照这个1rem=10的比例写出了静态页面，将设计图上的元素答谢全部用rem写下 4、这样你写的静态页面再640px上的页面是正常展示的！ 5、假设a用户使用的是375px宽（设备独立像素，浏览器调试工具所展示的），因为你假设75rem=100%，因此此时的1rem=5px（375 / 750 * 10px），这样用户在375宽度的手机上也完美运行了

    # 2、rem怎么用

    1、rem是css单位 2、1rem的大小是通过html下的font-size这个属性告诉浏览器的 3、替换px所在的位置即可

    预设比例：1rem = 10px （n个rem等于100%的可视宽，为了好算通常比例为10.几等份根据设计稿确认） 假设一个图片img（class=’img‘）宽度为60px,高度为50px。（设计稿位图像素） 那么css可以写成：

    html{
    font-size:10px;
}
.img{
    width:6rem;
    heighy:5rem
}

    # 3、适配任何的分辨率

    如何确定1rem到底等于多少px饿 css是不会帮你自动算的，只能通过自己去计算 计算规则： 1、设计稿位图像素宽A（750px），然后预设将设计稿等分B份（A/10 =75），也就是1rem=10px， 2、获取用户当前的可是宽度C（1、可以通过js动态获取；2、通过媒体查询预设方案）。那么他的1rem对应碓尺寸就是C/B，也就是将当前碓设备等分为B份对应每份的px数 3、假设A=750px，B=75，C=375px ==》 1rem = 375/75

    动态设置当前用户的rem

    var fun = function (doc, win) {
    var docEl = doc.documentElement,
        resizeEvt = 'orientationchange' in window ? 'orientationchange' : 'resize',
        recalc = function () {
            var clientWidth = docEl.clientWidth;
            if (!clientWidth) return;

            //这里是假设在640px宽度设计稿的情况下，1rem = 20px；
            //可以根据实际需要修改
            docEl.style.fontSize = 20 * (clientWidth / 640) + 'px';
        };
    if (!doc.addEventListener) return;
    win.addEventListener(resizeEvt, recalc, false);
    doc.addEventListener('DOMContentLoaded', recalc, false);
}
fun(document, window);

    # 二、相关知识大杂烩

    # 1、视觉稿相关

    psd格式图：ui根据原型图和业务要求设计
    多倍图：为了页面清晰，视觉稿规范

    • 首先选取一种手机尺寸作为基准，一般采用 iphone6 的 375x667 （设备独立像素，密度无关像素）
    • 对于retia屏的dpr=2，为了高清显示，视觉稿的画布会设计成基准的两倍，即两倍图，像素点数是原来的两倍

    二倍图：

    • 1个位图像素对应1个物理像素,图片才是完美清晰的展示
    • 在高分屏时如dpr=2的1retina屏,1个位图像素对应4个物理像素
      • 单个位图像素不能再分割,只能就近取色，导致图片模糊
    • 解决方法：ui提供2倍图
      • 在dpr=2的屏幕上,1个位图像素对应一个物理像素，图片显示清晰
      • 在普通屏幕，缩减像素采样，Downsampling

    dpi：dpi是单位，一般指每英寸的像素。（相当于密度），200dpi的图片，假设长10英寸，宽5英寸。那么分辨率就是2000*1000。要注意，图片分辨率与电脑屏幕尺寸的分辨率不一样，它不是尺寸大小，只是指精细度。

    # 2、像素

    物理像素： 一个物理像素是显示器最小的物理显示单位

    位图像素：栅格图像(png,jpg,gif等)最小的数据单位，每一个位图像素都包含一些自身的信息（显示位置、颜色值、透明度等）

    设备独立像素（密度无关像素）：

    • 计算机系统中的一个点（单位），这个点代表能够被程序员使用点虚拟像素（css像素等）。可以根据dpr转换为物理像素
    • 设备独立像素点一个单位：dp（android开发），pt（ios开发），css（web移动开发）

    设备像素比（dpr）：

    • =物理像素/设备独立像素 ，在某一方向上，x或y
    • 获取：
      • js中： window.devicePixelRatio
      • css中：媒体查询预设规范

    # 3、 屏幕

    屏幕尺寸：主屏尺寸，他的单位是英尺（in),对角线尺寸

    屏幕分辨率：对应的物理像素

    屏幕像素密度（ppi）：

    • 对角线的无聊像素/屏幕尺寸
    • PPI(Pixels(像素) Per(每) Inch(英寸))
    • 在相同屏幕尺寸下PPI值越大,那么所能显示的图像越清晰

    # 4、长度单位

    相对长度单位

    • 相对长度单位：就是会随着单位的使用的地方不同，其值会发生变化。比如:em px(只在表示设备分辨率的时候) %

    • 绝对长度单位：绝对长度单位:不会随着使用环境的变化而变化，值是绝对的，比如:in(英寸) cm(厘米) mm(毫米)

    # 5、视口 这个

    概念：视口是用来约束网站中最顶级块元素(html)的,也就是它决定了html的大小
    视口点作用：

    视口-->html大小-->页面布局

    • pc端viewport的大小和浏览器窗口大小一致,因此我们不会去管他
    • 移动端屏幕较小,视口足够大的化的html也会变大,pc端的网页也会在移动端较为完全的显示;出现滚动条

    详解视口

    • layout viewport(视窗布局视口)，网页布局区域的大小(以CSS像素为单位)，大多数的默认的layout是980px
    • ideal viewport(理想视口)，大小就是设备屏幕区域(以设备独立像素dp、pt为单位)

    视口设置

    • 移动端理想状态：避免滚动条且不被默认缩放处理
    <meta name="viewport" content="width=device-width,initial-scale=1.0">
    1
    • 其他设置

    initial-scale 设置页面的初始缩放值(倍数)，为一个数字
    maximum-scale 允许用户的最大缩放值(倍数)，为一个数字
    minimum-scale 允许用户的最小缩放值(倍数)，为一个数字
    user-scalable 否允许用户进行缩放，值为"no"或"yes", no 代表不允许，yes代表允许

    # 6、rem和em

    rem

    • 永远等于根节点html的font-size(基准值)

    em

    • 根据父节点的font-size(基准值)

    # 三、移动端开发相关

    移动端布局方案：

    1、固定布局：很早以前其实就用的固定宽度布局的方案,只不过现在已经被废弃了
    2、百分比布局：流式布局，特别适合pc端布局
    3、rem布局：用rem这个单位来做不同手机的适配，现在做手机端页面最好的

    移动端浏览器

    • 手机自带浏览器

    # 四、媒体查询

    # 什么是媒体查询

    • 媒体查询是CSS3的中一个强大的工具。它是包含媒体类型以及零或多个检查特定媒体特征条件的表达式。
    • 表达式描述了媒体特征, 最终会被解析为 true 或 false.如果是true 那么会执行 如果是false 就不会执行了.
    • 要做响应式(媒体查询) 还要做rem布局(需要用到媒体查询)
    • 通过不同的判别条件识别不同的设备 以给该设备应用不同的样式

    # 媒体特性

    width 视口的宽度
    height 视口的高度
    device-width 设备屏幕宽度
    device-height 设备屏幕高度
    orientation 横屏竖屏
    portrait （纵向）
    landscape（横向）
    width / height
    max-width / max-height

    1、(最大宽度==小于等于多宽)/(最大高度==小于等于多高)
    2、指的是显示区域的宽度/高度，比如浏览器的显示区域宽度/高度
    3、max-width在窗口大小改变或横竖屏转换时会发生变化

    min-width / min-height

    指的是设备整个渲染（显示）区域的宽度，比如设备的实际屏幕大小，也就是屏幕分辨率
    max-device-width只与设备相关，横竖屏转换或改变尺寸，缩放都不会发生变化（部分android的宽高会互换而IOS不会）

    device-width / device-height

    这两个值是从 chrome浏览器的移动模拟器中拿到的值
    device- width指的是设备本身的宽度，也就是屏幕的分辨率

    orientation

    portrait | landscape 横 | 竖
